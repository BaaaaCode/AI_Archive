# 🚀 실험용 한국어 RAG 챗봇 (Educational RAG Chatbot)

이 프로젝트는 **비전공자 및 AI 입문자**를 위해 만들어진 **한국어 RAG(검색 증강 생성) 챗봇** 실습 예제입니다.
복잡한 수식이나 이론보다는, **"내 데이터를 AI에게 가르쳐서 대답하게 만드는 원리"**를 이해하는 데 초점을 맞추었습니다.

---

## 📚 핵심 개념 (이것만은 알고 가요!)

이 프로젝트에서 사용된 4가지 핵심 기술을 소개합니다.

### 1. RAG (Retrieval-Augmented Generation, 검색 증강 생성)
AI(LLM)는 똑똑하지만, **우리가 가진 비공개 문서(사내 규정, 내 일기장 등)**의 내용은 모릅니다.
RAG는 시험 칠 때 **"오픈북 테스트"**를 하는 것과 같습니다. AI에게 질문만 던지는 것이 아니라, **질문과 관련된 참고 자료(Book)를 같이 건네주며 답하게 하는 기술**입니다.
> **과정**: 질문 입력 -> 관련 문서 찾기(Search) -> 문서와 질문을 AI에게 전달 -> 답변 생성

### 2. 임베딩 (Embedding) & 벡터 DB (ChromaDB)
컴퓨터는 한글이나 영어를 이해하지 못합니다. 그래서 **글자를 숫자(좌표)로 바꾸는 작업**이 필요한데, 이를 **임베딩**이라고 합니다.
비슷한 뜻을 가진 단어들은 숫자 공간에서도 서로 가까운 곳에 위치하게 됩니다.
* 예: '사과'와 '배'는 가깝고, '사과'와 '자동차'는 멉니다.
**ChromaDB**는 이렇게 숫자로 변환된 데이터들을 저장해두었다가, 질문이 들어오면 **"숫자적으로 가장 가까운(가장 관련있는) 내용"**을 빠르게 찾아주는 도서관 역할을 합니다.

### 3. 형태소 분석 (Kiwi)
한국어는 '했습니다', '했는데', '하시니' 처럼 어미 변화가 심합니다.
AI가 문맥을 더 잘 찾을 수 있도록, 문장을 **가장 작은 의미 단위(형태소)**로 쪼개주는 작업입니다.
이 프로젝트에서는 **Kiwi**라는 도구를 사용해 한글을 깔끔하게 정리했습니다.

### 4. LLM (Google Gemini)
실제로 우리에게 답변을 해주는 **인공지능 두뇌**입니다.
우리가 찾아낸 자료를 바탕으로, 사람처럼 자연스러운 문장을 만들어줍니다. 여기서는 Google의 **Gemini Pro** 모델을 사용했습니다.

## 🛠️ 설치 및 실행 방법

### 1. 환경 설정
파이썬(Python)이 설치되어 있어야 합니다. 터미널(명령 프롬프트)에서 아래 명령어를 입력해 필요한 도구들을 설치합니다.

```bash
pip install -r requirements.txt
```

### 2. 실행
설치가 끝났다면, 아래 명령어로 챗봇 웹사이트를 실행합니다.

```bash
streamlit run app.py
```

### 3. 사용법
1.  브라우저가 열리면 사이드바에 **Google API Key**를 입력하고 **[저장]** 버튼을 누릅니다.
2.  **[DB 구축하기]** 버튼을 눌러 `data` 폴더에 있는 문서들을 학습시킵니다.
    * `data` 폴더에 원하는 `.txt` 파일을 넣으면 그 내용을 공부합니다!
3.  채팅창에 궁금한 내용을 물어보세요. (예: "전자기록물 폐기 절차가 뭐야?")
4.  답변 아래 **[📚 참조 문서]**를 눌러보면, AI가 어떤 내용을 참고했는지 확인할 수 있습니다.

## 📂 프로젝트 구조

*   `app.py`: 프로그램의 **본체**입니다. 화면 구성, AI 로직 등이 모두 여기에 있습니다.
*   `data/`: AI에게 학습시킬 **텍스트 파일(.txt)**들을 넣어두는 곳입니다.
*   `requirements.txt`: 이 프로젝트에 필요한 **도구 목록**입니다.
*   `.env`: API Key 같은 **비밀번호**가 저장되는 파일입니다. (남들에게 보여주지 마세요!)
*   `antigravity_db/`: 문서 내용이 **숫자(벡터)**로 변환되어 저장되는 폴더입니다.

## 📝 팁 (Tip)
*   **"제공된 문서에서 내용을 찾을 수 없습니다"**라고만 하나요?
    *   질문과 관련된 내용이 문서에 없거나, 너무 적게 포함되어 있어서 그렇습니다. 내용을 보충하거나 검색 범위(k)를 늘려볼 수 있습니다.
*   **파일을 수정했다면?**
    *   `data` 폴더의 파일을 수정했다면, 반드시 **[DB 구축하기]** 버튼을 다시 눌러서 업데이트해야 합니다.
